Introduction

初始：

容器化技术

容器技术之前，都使用虚拟机技术VMware，通过软件VMware可以虚拟出来一台或多台电脑。笨重。

docker，隔离，镜像（Core：4m+jdk+mysql），运行镜像即可。小巧。

虚拟机也属于虚拟化技术；Docker容器技术也是一种虚拟化技术。

Go:

Docker基于Go开发，开源

Docker can do how？

之前的虚拟机技术缺点：

  • 资源占用多
  • 冗余步骤多
  • 启动很慢

容器化技术

并非一个完整的操作系统

  • Docker应用直接运行在宿主机的内核，没有自己的内核
  • 每个容器间相互隔离

DevOps (开发、运维) 中级程序员 –> 高级程序员

应用更快速的交付和部署

Docker：打包镜像发布测试，一键运行

更便捷的升级和扩缩容

项目打包为一个镜像

更简单的系统运维

容器化之后，开发和测试环境高度一致。

Docker的基本组成

  • 镜像（image）：通过镜像可以创建多个容器。
  • 容器（container）：利用容器技术，Docker独立运行一个或一组应用。 基本命令：启动，停止，删除。
  • 仓库（repository）：存放镜像的地方。分为公有和私有仓库。 Docker Hub

Docker 底层架构

bootfs(boot file system) 主要包含bootloader 和 kernel。类似于操作系统：Ubuntu， Centos等。

一个OS的rootfs可以很小，只需包含基本命令+工具+程序库即可。因为底层使用的是宿主机Host的kernel。

所以！！下载镜像的时候是一层一层下载的！！

!! Docker 的镜像都是只读的，容器被放到镜像的顶部用于可写层！

所以顶层 -> 容器层， 下面的层为镜像层！！

容器数据卷

What’s this?
docker 理念回顾

将应用和环境打包成一个镜像

problem：数据都在容器中，容器删除则数据丢失！

需求：数据可持久化

eg: MySQL的数据可以存储在本地。

容器之间有一个可以数据共享的技术。docker容器中产生的数据同步到本地！

即卷技术：目录的挂载，将容器内的目录挂载到Linux上。

target：容器的持久化和同步操作；容器间数据共享。
